# Contributing

Thanks for your interest in Stitch!

## Quick start

- Open an issue to discuss changes before starting major work.
- Keep pull requests focused and small.
- Follow the existing code style; prefer clarity over cleverness.

## Development notes

- This project is static HTML/CSS/JS. No build step is required.
- Please test UI changes in a modern browser and on a narrow viewport.

## Submitting a PR

1. Fork the repo and create a branch.
2. Make your changes.
3. Update documentation if needed.
4. Open a PR with a concise description and screenshots for UI updates.
